# 推荐科研写作工作流 / Recommended Research-Writing Workflow

## 1. 建立事实底座 / Build the evidence base

- 固定当前论文母版、代码版本、实验表格和参考文献库。
- 建立术语表、符号表、贡献清单和待验证问题清单。
- 对每个结论记录证据来源：代码、实验、论文原文或作者解释。

## 2. 论文结构审计 / Structural audit

- 任务与问题是否明确。
- 现有方法的不足是否能自然导出本文方案。
- 每项贡献是否有方法模块和实验结果对应。
- 章节之间是否存在重复动机、重复定义或结论超出证据。

## 3. 原子化修改 / Atomic revision

每轮只修改一个最小章节：

1. 说明本轮范围与不可修改内容。
2. 列出当前问题和修改目标。
3. 给出完整替换稿。
4. 执行术语、符号、公式、图表、引用和字数检查。
5. 把跨章节问题加入待处理清单，不在本轮越界修改。

## 4. 引用与事实核验 / Citation and claim verification

- 每条引用都应支持它前面的具体主张。
- 不通过论文标题猜测内容。
- 区分预印本、会议版和期刊扩展版。
- 表格数字应能追溯到原始实验输出。

## 5. 投稿前多角色QC / Multi-role pre-submission QC

依次执行：

- 领域审稿人：创新性与技术正确性。
- 逻辑编辑：问题—方法—证据闭环。
- 数学审校：符号唯一性、维度、公式编号和定义顺序。
- 实验审校：数据、指标、基线、公平性和统计结论。
- 引用审校：真实性、位置和格式。
- 语言编辑：准确、简洁、避免模板化表达。

## 6. 作者最终确认 / Final author sign-off

作者必须确认：所有实验真实运行、所有引用真实阅读、所有结论不超过证据、所有AI修改均被理解和接受。
